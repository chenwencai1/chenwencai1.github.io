双向数据绑定的实现原理

  • Object.defineProperty
  • 实现的底层原理


Vue 主要通过以下 4 个步骤来实现数据双向绑定的：

  • 实现一个监听器 Observer：对数据对象进行遍历，包括子属性对象的属性，利用 Object.defineProperty() 对属性都加上 setter 和 getter。这样的话，给这个对象的某个值赋值，就会触发 setter，那么就能监听到了数据变化。
  • 实现一个解析器 Compile：解析 Vue 模板指令，将模板中的变量都替换成数据，然后初始化渲染页面视图，并将每个指令对应的节点绑定更新函数，添加监听数据的订阅者，一旦数据有变动，收到通知，调用更新函数进行数据更新。
  • 实现一个订阅者 Watcher：Watcher 订阅者是 Observer 和 Compile 之间通信的桥梁 ，主要的任务是订阅 Observer 中的属性值变化的消息，当收到属性值变化的消息时，触发解析器 Compile 中对应的更新函数。
  • 实现一个订阅器 Dep：订阅器采用 发布-订阅 设计模式，用来收集订阅者 Watcher，对监听器 Observer 和 订阅者 Watcher 进行统一管理。

4.vue的单向数据流

所有的 prop 都使得其父子 prop 之间形成了一个单向下行绑定：父级 prop 的更新会向下流动到子组件中，但是反过来则不行。

Vue 的父组件和子组件生命周期钩子函数执行顺序可以归类为以下 4 部分：

  • 加载渲染过程：父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted
  • 子组件更新过程：父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated
  • 父组件更新过程：父 beforeUpdate -> 父 updated
  • 销毁过程：父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

每次父级组件发生更新时，子组件中所有的 prop 都将会刷新为最新的值。这意味着你不应该在一个子组件内部改变 prop。如果你这样做了，Vue 会在浏览器的控制台中发出警告。子组件想修改时，只能通过 $emit 派发一个自定义事件，父组件接收到后，由父组件修改。有两种常见的试图改变一个 prop 的情形 :

  • 这个 prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用。 在这种情况下，最好定义一个本地的 data 属性并将这个 prop 用作其初始值
  • 这个 prop 以一种原始的值传入且需要进行转换。 在这种情况下，最好使用这个 prop 的值来定义一个计算属性

面向对象之类的继承

面向对象之类的继承

知识铺垫

封装

低耦合高内聚

多态

  • 重载：方法名相同，但是形参个数和类型不一样（JS不存在重载，JS重载重载指的是同一个方法，根据传参不同实现不同的功能效果）

  • 重写：在类的继承当中，子类可以重写父类中的方法，返回值和形参都不能改变。

    面向对象的理解

  • 面向对象是一种编程思想。JS本身就是基于面向对象构建出来的（例如：JS当中有很多内置类，向Promise就是ES6中一个新增的内置类，我们可以基于new Promise来创建一个实例，来管理异步编程，我在项目中Promise也经常用，自己也看过他的源码。。。），我们平时用的VUE/REACT/JQUERY也是基于面向对象构建出来的，他们都是类，平时开发的时候都是创建他们的实例来操作的。。。

  • JS中的面向对象，和其它编程语言还是有略微不同的，JS中类和实例是基于原型和原型链机制来处理的；而且JS中关于类的重载、重写、继承也和其它语言不太一样…

    继承：子类继承父类上的属性和方法

    目的：让子类的实例能够使用父类中的属性和方法

    原型继承

  • 继承方式：让父类中的属性和方法在子类的实例的原型上
    child.prototype=new parent();
    child.prototype.constructor=child;

  • 原型继承特点：

    1. 不像其他语言中的继承一样（其他的语言的继承一般是拷贝继承，也就是子类继承父类，会把父类中的属性和方法拷贝一份到子类中，供子类的实例调用），他是把父类的原型放到子类的原型链上，子类想调用这些方法是基于proto原型链机制查找继承的

    2. 子类可以重写父类上的方法，这样会导致父类上其他的实例也会受到影响

    3. 父类的私有或者公有的属性和方法都会编成子类中公有的属性和方法

call继承

  • 继承方式：child方法中把parent方法当做普通函数执行，让parent中的this指向child的实例，
  • call继承的特点：
    1. 只能继承父类私有的属性和方法（因为是把parent当做普通函数执行，和其原型上的属性和方法没有关系）
    2. 父类私有的变为子类私有的

寄生组合继承：call继承+类似于原型继承

  • 继承方式：创建一个空对象，将空对象的原型指向父类的原型，将这个空对象作为子类对象原型
  • 寄生组合继承特点：父类的私有和公有的分别是子类实例的私有和公有属性方法（推荐使用）

ES6中的继承处理

  • 语法：
    class child extends parent{}相当于实现child.prototype.\proto__=parent.prototype

    • 子类继承父类可以不写construtor，如果写了就必须先写super(),

  • class类的特点：

    1. ES6中基于CLASS创造出来的类不能当做普通函数执行,只能new执行
    2. ES6中基于class创建出来的类不允许重定向原型的指向

DOM0事件绑定和DOM2事件绑定

DOM0和DOM2操作dom的版本

DOM0

元素.on事件行为=function(){}

DOM0事件绑定原理(只能绑定一个函数)

给元素的私有属性（事件行为）赋值（就是绑定的函数），当事件触发时，浏览器会帮我们把赋值的函数执行。但是这样同一个元素相同的事件行为绑定一个方法，重复绑定时，触发时以最后一个方法为主执行

DOM0事件绑定移除

元素.on事件行为=null; (根据绑定原理移除)

DOM2（DOM2事件绑定的时候，一般采用实名函数，目的：这样可以基于实名函数移除事件绑定）

元素.addEventListener(事件行为，function fn(){},true/false);

DOM2事件绑定原理（可以绑定多个函数）

基于原型链查找机制，找到EventTarget.prototype上的方法并且执行，此方法执行，会把给当前元素某个事件行为绑定的所有方法，存放在浏览器某人的事件池中（绑定几个方法，会向事件池存储几个），当事件触发一次，会把事件池中存储的对应方法，依次按照顺序执行。给当前元素某一个事件行为绑定多个不同的方法,同一个元素行为绑定多个同一个方法会默认去重

DOM2事件绑定移除

事件对象 ev

给元素的事件行为绑定方法，当事件行为触发，方法被执行，不仅被执行，而且还会把当前操作的相关信息传递给这个函数。=>传递的信息就是事件对象
事件对象和函数以及给谁绑定的事件没有关系，它存储的是当前本次操作相关的信息，操作一次只能有一份信息，所以在哪个方法中获取的方法信息是一样的；第二次操作，存储的信息会把上一次的操作的信息替换掉。
每一次事件触发，浏览器都会这么做

  1. 捕获到当前操作的行为（把操作的信息获取到），通过创建MouseEvent等类的实例，得到事件对象EV
  2. 通知所有绑定的方法，（符合执行条件）

事件对象类型

  1. 鼠标事件对象：鼠标操作，获取的是MouseEvent类的实例
    鼠标事件对象->MouseEvent.prototype->UIEvent.prototype->Event.prototype->Object.prototype
  2. 键盘事件对象：键盘操作，获取的是KeyboardEvent类的实例。
  3. 普通事件对象：获取的是Event类的实例
  4. 手指事件对象：获取的是TouchEvent类的实例

鼠标事件对象 语法：ev.事件对象

  1. clientX/clientY:当前鼠标触发点距离窗口左上角的X/Y轴坐标
  2. pageX/Y:鼠标触发点距离当前页面左上角的X/Y轴坐标。
  3. type：当前触发事件的类型
  4. target：事件源（操作的目标元素就是事件源）
  5. preventDefault():阻止默认行为的方法，不兼容的浏览器中庸ev.returnValue=false,也可以的实现阻止默认行为
  6. stopPropagation()：阻止冒泡传播，不兼容的浏览器中用ev.cancelBubble=true也可以阻止默认行为。

事件的传播机制

  1. 捕获阶段：从最外层向最里层事件源依次查找（目的：是为冒泡阶段事先计算好传播的层级路径）
  2. 目标阶段：当前元素的相关事件行为触发。
  3. 冒泡传播机制：触发当前元素的某一个事件行为，不仅本身的行为被触发了，而且所有的祖先元素（一直到window）相关的事件行为依次都会被触发（从内到外的顺序）
    stopPropagation()：阻止冒泡传播

DOM0绑定的事件只能在目标阶段和冒泡阶段触发执行（也有捕获阶段）；DOM2绑定的方法，我们可以控制在捕获阶段执行
false代表在冒泡阶段执行，默认为false，true代表在捕获阶段执行此方法（用的很少）

AXIOS基础知识

AXIOS类库的应用

  • 一款基于Promise设计模式封装的AJAX库

    options

  • 在使用AXIOS之前，我们一般都需要配置默认的配置项

  1. baseURL：基础的URL.后期再发送请求的时候，URL请求地址最前面的公共部分就不需要再写了

  2. transformRequest：处理请求参数（对post系列有作用）.设置请求拦截器（只对POST系列有用）：把基于请求主体传递给服务器的内容进行拦截，把内容格式变为x-www-form-urlencoded这种格式，再传递给服务器(JQ中默认传递了)

  3. transformResponse：处理返回的结果进行处理.设置响应拦截器：[成功状态]把从服务器获取的结果中的响应主体信息获取到即可，[失败状态]手动把错误信息抛出异常

  4. headers：设置请求头:POST系列中，我们传递给服务器数据的格式一般以x-www-form-urlencoded格式为主

  5. params：get系列请求传递给服务器的内容（会把Params中的内容拼接为x-www-form-urlencode这种格式，基于url问号传参传递给服务器）(JQ中默认清缓存，axios要手动去缓存)

  6. paramsSerializer：传递参数的序列化

  7. timeout：超时时间

  8. withCedentials：跨域请求中是否允许携带凭证

  9. responseType：预设服务器返回结果的格式：默认是json，支持buffer、text、stream、decument

  10. validateStatus：axios本身只有在HTTP状态码以2开头的时候才认为是成功的，其余都认为是失败状态，当然我们可以基于validateStatus这个自己来修改。

Promise基础知识

Promise

  • ES6语法中新增加的内置类，用来处理JS中异步编程的。
  • Promise设计模式：基于promise对异步操作进行管理

    异步编程中的回调地狱（都是异步）

  • ajax的串行：请求是一个一个发送的，上一个请求完成后才进行下一个请求，每次请求成功后拿到此次请求的数据，最后一个请求成功后就拿到每一次请求的数据
    ajax里面再次嵌套ajax执行
  • ajax的并行：多请求可以同时发送，但是需要所有请求都成功才可以做某些事情。

    Promise基础语法

    Promise实例的三种状态PromiseStatus

  • pending：初始状态（new Promise（）后的状态）
  • fulfilled：成功状态（在executor函数中把resolve执行，就是告诉Promise当前异步操作的结果是成功的，将状态由pending改为fulfilled）
  • rejected：失败状态（在executor函数中把reject执行，就是告诉Promise当前异步操作的结果是失败的的，将状态由pending改为rejected）

    PromiseValue:resolve函数、reject函数中传递的参数

  • resolve函数或者reject函数执行后可以改变PromiseValue的值
  • PromiseValue一旦被改变，立刻通知then方法，执行相应的方法。所以状态只可能改变一次。

    执行方式： new Promise([executor])

  • 执行函数作为参数函数必须传递
  1. new Promise的时候执行executor函数，并创建一个promise实例（executor是Promise类的一个回调函数，Promise内部会把它执行）
  2. Promise不仅把executor执行，还会给执行（executor）函数传递两个参数，这两个参数也是函数类型的：
    • resolve函数：它执行代表Promise处理的异步事情是成功的，把Promise的状态改为fulfilled
    • reject函数：它执行代表Promise处理的异步事情是失败的，把Promise的状态改为rejected
  3. executor函数中放的就是当前要处理的异步操作
  4. resolve函数或者reject函数执行后根据resolve函数或者reject函数的参数可以改变PromiseValue的值

    Promise.prototy实例上三个的方法

    then方法：

  • 向事件池中设置成功或者失败后执行的方法，两个都可以设置，也可以只设置一个
  • 基于then方法，构建成功或者失败后要做的事情。当实例状态执行后，状态改变，通知then方法，传递相应的参数执行对应的方法

    catch方法：

  • 向事件池中设置失败后执行的方法

    finally方法：

  • 设置不论成功还是失败都会执行的方法（一般不用）
