常用的事件行为

  • JS

事件是元素天生自带的默认行为。

常用的事件行为

鼠标事件

  1. click 点击 （移动端click被识别为单击）
  2. dblclick 双击
  3. mousedown 鼠标按下
  4. mouseup 鼠标抬起
  5. mouseover 鼠标滑过
  6. mouseout 鼠标滑出
  7. mouseenter 鼠标进入
  8. mouseleave 鼠标离开
  9. mousewhell 鼠标滚轮滚动

键盘事件

  1. keydown 键按下
  2. keyup 抬起某个键
  3. keypress 除shift/FN/capsLock键以外，其他键按住连续触发
  4. contextmenu 鼠标右键触发

移动端手指事件

单手指事件模型 Touch

  1. touchstart 单手指按下
  2. touchmove 单手指移动
  3. touchend 单手指松开
  4. touchcancel 操作取消（一般应用于非正常状态下的操作结束：例如手机突然关机）

多手指事件模型 Gesture

  1. gesturestart 多手指按下（会记录多有按下手指的信息）
  2. gesturechange/gestureupdata
  3. gestureend
  4. gesturecancel

表单元素常用事件

  1. focus 获取焦点
  2. blur 失去焦点
  3. change 内容改变

音视频常用事件

  1. canplay 可以播放（资源没有加载完，播放中可能会卡顿，边播放边加载）
  2. canplaythrouht 可以播放（资源已经加载完，播放中不会卡顿）
  3. play 开始播放
  4. playing 播放中
  5. pause 暂停播放

其他事件

  1. load 加载成功事件
  2. unload 文档或者一个资源被卸载
  3. beforeunload 当前页面关闭之前
  4. error 资源加载失败
  5. scroll 滚动事件
  6. readystatechange AJAX请求状态改变事件

DOM0事件绑定和DOM2事件绑定

DOM0和DOM2操作dom的版本

DOM0

元素.on事件行为=function(){}

DOM0事件绑定原理(只能绑定一个函数)

给元素的私有属性（事件行为）赋值（就是绑定的函数），当事件触发时，浏览器会帮我们把赋值的函数执行。但是这样同一个元素相同的事件行为绑定一个方法，重复绑定时，触发时以最后一个方法为主执行

DOM0事件绑定移除

元素.on事件行为=null; (根据绑定原理移除)

DOM2（DOM2事件绑定的时候，一般采用实名函数，目的：这样可以基于实名函数移除事件绑定）

元素.addEventListener(事件行为，function fn(){},true/false);
JQ中元素.on(事件行为，function fn(){},true/false)
IE6~8中：元素.attachEvent(‘on事件行为’，function(){});

DOM2事件绑定原理（可以绑定多个函数）

基于原型链查找机制，找到EventTarget.prototype上的方法并且执行，此方法执行，会把给当前元素某个事件行为绑定的所有方法，存放在浏览器某人的事件池中（绑定几个方法，会向事件池存储几个），当事件触发一次，会把事件池中存储的对应方法，依次按照顺序执行。给当前元素某一个事件行为绑定多个不同的方法,同一个元素行为绑定多个同一个方法会默认去重

DOM2事件绑定移除

移除事件绑定：从事件池中移除，多以需要指定好事件类型、方法信息（要和绑定的时候一样才可以移除）
元素.removeEventListener(事件行为，func ,false);
JQ中 元素.off(事件行为，func ,false);

dom0 和dom2 同时出现时，执行顺序

  1. 机制不同，不会有影响，按绑定顺序执行。
  2. dom0能做的事件绑定的事件行为，dom2都支持，dom2里面的事件，dom0不一定能处理绑定，例如：transitionend、DOMContentLoaded。。。
    DOMContentLoaded：DOM结构加载完就可以触发。
    $(function(){})和$(document).ready(function(){})这个方法就是采用的 DOMContentLoaded事件，并且依托DOM2事件绑定来处理，所以同一个页面中，此操作可以被使用多次
    window.onload和$(document).ready()的区别
    jQuery中的事件绑定采用的都是DOM2的事件绑定
    load：所有的结构加载完才触发。

事件对象 ev

给元素的事件行为绑定方法，当事件行为触发，方法被执行，不仅被执行，而且还会把当前操作的相关信息传递给这个函数。=>传递的信息就是事件对象
事件对象和函数以及给谁绑定的事件没有关系，它存储的是当前本次操作相关的信息，操作一次只能有一份信息，所以在哪个方法中获取的方法信息是一样的；第二次操作，存储的信息会把上一次的操作的信息替换掉。
每一次事件触发，浏览器都会这么做

  1. 捕获到当前操作的行为（把操作的信息获取到），通过创建MouseEvent等类的实例，得到事件对象EV
  2. 通知所有绑定的方法，（符合执行条件）

事件对象类型

  1. 鼠标事件对象：鼠标操作，获取的是MouseEvent类的实例
    鼠标事件对象->MouseEvent.prototype->UIEvent.prototype->Event.prototype->Object.prototype
  2. 键盘事件对象：键盘操作，获取的是KeyboardEvent类的实例。
  3. 普通事件对象：获取的是Event类的实例
  4. 手指事件对象：获取的是TouchEvent类的实例

鼠标事件对象 语法：ev.事件对象

  1. clientX/clientY:当前鼠标触发点距离窗口左上角的X/Y轴坐标
  2. pageX/Y:鼠标触发点距离当前页面左上角的X/Y轴坐标。
  3. type：当前触发事件的类型
  4. target：事件源（操作的目标元素就是事件源）
  5. preventDefault():阻止默认行为的方法，不兼容的浏览器中庸ev.returnValue=false,也可以的实现阻止默认行为
  6. stopPropagation()：阻止冒泡传播，不兼容的浏览器中用ev.cancelBubble=true也可以阻止默认行为。

点击A标签，先执行click事件，再执行href的跳转

键盘事件对象

  1. code/kry:存储的都是按键，code更细
  2. keyCode&which:储存的是键盘按键上的码值。
    方向键：上下左右键分别是37、38、39、40
    空格键space：32
    回车键：enter13
    删除键：del46
    后退键：back8
    ALT：18
    ctrl：17
    shift：16

阻止默认行为三种方式

  1. javascript:;
  2. 在绑定事件中返回false
  3. 使用preventDefault()

事件的传播机制

  1. 捕获阶段：从最外层向最里层事件源依次查找（目的：是为冒泡阶段事先计算好传播的层级路径）
  2. 目标阶段：当前元素的相关事件行为触发。
  3. 冒泡传播机制：触发当前元素的某一个事件行为，不仅本身的行为被触发了，而且所有的祖先元素（一直到window）相关的事件行为依次都会被触发（从内到外的顺序）
    stopPropagation()：阻止冒泡传播

DOM0绑定的事件只能在目标阶段和冒泡阶段触发执行（也有捕获阶段）；DOM2绑定的方法，我们可以控制在捕获阶段执行
false代表在冒泡阶段执行，默认为false，true代表在捕获阶段执行此方法（用的很少）

mouseenter和mouseover的区别

  1. mouseover:本身不是进入，而是看光标在谁的上面，从子元素进入父元素，触发父元素的mouseover，从父元素进入子元素会触发父元素的mouseout。。。
