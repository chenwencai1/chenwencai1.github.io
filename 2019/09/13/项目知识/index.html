项目结构目录

  • admin：存储所有的后台程序
  • client：客户端所有的程序

    admin

  1. 首先跑环境 $ npm i
  2. 纠正一下基础的配置：config.js
    • port:后台程序运行的端口
    • cros.allow_origin: 客户端程序运行的服务地址
  3. 启动服务：$ node server.js
  4. 启动后的运行窗口不能关闭，关闭则后台服务器关闭

    client

    客户端采用的技术栈：
  5. HTML5、CSS3（会用到字体图标）
  6. jQuery
  7. AXIOS
  8. 辅助插件：jQuery.cookie/MD5/自己写的一些方法
  9. 基于iframe构建最初版本的SPA单页面应用开发
    服务端采用的技术栈：
  10. node
  11. express/express-pouter
  12. 本应基于数据库存储，临时采用json文件存储数据
  13. session

    本地存储 VS 服务器存储

    本地存储：把信息存储在客户端本地。

    谷歌控制台Application中都可以看到
  • cookie
  • H5中webStorage：localStorge/sessionStorage
  • 本地数据库存储：IndexDB
  • 本地缓存存储：manifest
  • …

    本地存储使用场景

  1. 记住用户名和密码（或者自动登录）
  2. 未登录状态下，加入购物车的信息一般也先存储到本地，当登录后再把信息存储到服务器上，实现多平台数据共享
  3. 对应非实时刷新数据，我们可以在从服务器把数据获取到后，临时储存到本地（设置有效时间，在有效时间内内页面刷新，不再重新从服务器中获取信息，而是读取本地信息，超多有效时间，重新从服务器中获取信息）“前端数据优化的一点”
  4. 实现同一个网站不同页面之间的信息共享和通信

    本地存储的特点

  5. 本地存储是受浏览器限制的。不同浏览器之间不能信息共享
  6. 受源（域）的限制。例如，都是谷歌浏览器，在京东网站存储的数据，在百度网站中获取不到。
  7. 本地存储的信息在控制台可以看到，而且是明文存储，所以敏感的数据最好不要存储带本地，一定要存储到本地一定要做安全处理。（例如：加密）

    本地存储方法：

    向本地存储的信息都是string字符串格式的
  8. localStorage：持久化存储在客户端本地，除非手动清除或者浏览器卸载，否则一直存储下来，没有过期时间。
  • localStorage.setItem([key],[value]) 存储信息
  • localStorage.getItem([key]) 获取信息
  • localStorage.removeItem([key]) 移除信息
  • localStorage.clear() 清除所有存储的信息
  1. sessionStorage：会话存储（页面刷新存储页面还在，但是页面关闭后存储的信息就消失了）
  • sessionStorage.setItem([key],[value]) 存储信息
  • sessionStorage.getItem([key]) 获取信息
  • sessionStorage.removeItem([key]) 移除信息
  • sessionStorage.clear() 清除所有存储的信息
  1. cookie:
  • 设置、获取、移除cookie都是使用document.cookie=’age=25’;移除就是把cookit过期时间设置为已经过去的一个时间就可以了
  • 使用jQuery插件设置、获取、移除cookit
    cookie=>jquery.cookie.js
    $.cookie(“userName”,”陈文才”) 设置cookie
    $.cookie(“userName”) 获取cookie
    $.cookie(“userName”,”陈文才”,{
    expires:new Date(new Date().getTime()+302460601000)
    });

    cookie和localStorage的区别：

  1. 大小限制，一般浏览器允许一个源下cookie最多存储8kB。而localStorage被允许存储5MB
  2. 兼容问题。cookie兼容所有浏览器，而localStorage是H5 中新增的，不兼容IE低版本浏览器
  3. 稳定性。cookie有过期时间，但是一般不等到时间cookie可能就没有了，因为清除浏览器记录可以清除cookie，安全卫士清除也可以清除，而这些操作对localStorage没有影响。
  4. 浏览器开启无痕浏览或者隐私模式，此时无法设置cookie，但是可以设置localStorage的信息。
  5. 和服务器有联系。客户端和服务器端交互的时候，cookie信息会传来传去，而localStorage和服务器没有必然的联系，是单纯的本地存储
  6. 存储时间。cookie有过期时间，而localStorage是持久化存储。

  • redis:缓存数据存储

  • …
